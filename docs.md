# docs.md

#### о проекте

coredvd — это эмулятор dvd-плееров с собственным 32-битным процессором pulse.  
прошивки для pulse пишутся на ассемблере и компилируются в бинарный файл.  
в будущем в проекте появятся и другие процессоры (z80, arm и т.д.),  
но сейчас основная цель — показать, как работает программируемый dvd-плеер.

### процессор pulse

pulse - это наш risc-подобный 32-битный процессор с 16 регистрами (r0..r15).  
поддерживает флаг z (zero), который устанавливается после многих инструкций.  
стек растёт вниз, указатель стека - r14.  
программный счётчик (pc) - внутренний, не виден как регистр.

характеристики:
- 32-битные инструкции (переменной длины)
- прямая адресация памяти (до 64 кбайт озу)
- отдельная видеопамять (vram) для кадрового буфера

## система команд

все инструкции пишутся в нижнем регистре.  
аргументы разделяются запятыми.

список инструкций (опкод, синтаксис, действие):

### пересылка данных
- `mov rX, imm32` — загрузить 32-битную константу в регистр
- `movi rX, imm16` — загрузить 16-битную константу (little-endian)
- `load rX, addr32` — загрузить слово (4 байта) из озу
- `store addr32, rX` — сохранить слово в озу
- `ldrb rX, addr32` — загрузить байт
- `strb addr32, rX` — сохранить байт
- `ldrh rX, addr32` — загрузить полуслово (2 байта)
- `strh addr32, rX` — сохранить полуслово
- `push rX` — поместить регистр в стек
- `pop rX` — восстановить регистр из стека

### арифметика и логика
- `add rX, rY` — rX = rX + rY
- `sub rX, rY` — rX = rX - rY
- `mul rX, rY` — rX = rX * rY
- `div rX, rY` — rX = rX / rY (целочисленное)
- `mod rX, rY` — rX = rX % rY
- `and rX, rY` — побитовое и
- `or rX, rY` — побитовое или
- `xor rX, rY` — исключающее или
- `not rX` — побитовое отрицание
- `neg rX` — арифметическое отрицание (дополнительный код)
- `inc rX` — увеличить на 1
- `dec rX` — уменьшить на 1
- `cmp rX, rY` — сравнить (выставляет флаг z)

### сдвиги
- `lsl rX, rY` — логический сдвиг влево на rY бит
- `lsr rX, rY` — логический сдвиг вправо
- `asr rX, rY` — арифметический сдвиг вправо (знаковый)
- `ror rX, rY` — циклический сдвиг вправо

### управление потоком
- `jmp addr32` — безусловный переход
- `jz addr32` — переход, если флаг z = 1
- `jnz addr32` — переход, если флаг z = 0
- `call addr32` — вызов подпрограммы (адрес возврата в стеке)
- `ret` — возврат из подпрограммы
- `halt` — останов процессора

### системные и видео
- `print_char rX` — вывести на консоль символ из младшего байта регистра
- `load_frame` — показать текущий кадр из vob (специальная инструкция для dvd)
- `swi imm8` — системное прерывание (код 0 = выход, 1 = очистка экрана)
- `nop` — нет операции

больше подробностей прошивки в firmware/pulse/docs.md

## процессор z80

z80 - это классический 8-битный процессор, широко использовавшийся в компьютерах 80‑х (zx spectrum, amstrad cpc и др.).  
в coredvd он реализован в урезанном виде, достаточном для написания простых прошивок, управляющих воспроизведением видео.

### регистры

- **a** — аккумулятор, 8 бит (основной регистр для арифметики)  
- **b, c, d, e, h, l** — 8-битные регистры общего назначения  
- **флаги (f)** — 8-битный регистр флагов:  
  * z (zero) — результат равен нулю  
  * s (sign) — знак результата (старший бит)  
  * p/v (parity/overflow) — чётность или переполнение  
  * c (carry) — перенос/заём  
  * h (half-carry) — полуперенос (для bcd)  
  * n (add/subtract) — признак вычитания  
- **pc** — 16-битный программный счётчик  
- **sp** — 16-битный указатель стека (в текущей реализации не используется)

### особенности реализации в coredvd

- поддерживается только **подмножество инструкций** (см. таблицу ниже).  
- память: прошивка загружается по адресу `0x0000` (до 64 кбайт), но команды чтения/записи в ram (`ld (hl), a` и т.п.) **не реализованы**.  
- единственный способ вывести кадр на экран — вызвать псевдоинструкцию **`load_frame`** (кодируется как `rst 18`, опкод `0xdf`).  
- внутренний 32-битный счётчик кадров (`regs[15]`) автоматически увеличивается при каждом `load_frame`.  
- скорость воспроизведения, пауза, переключение глав — всё управляется с клавиатуры, прошивка только вызывает `load_frame` в цикле.

### поддерживаемые инструкции

| инструкция  | опкод | байт | действие |
|-------------|-------|------|-----------|
| `nop`       | 0x00  | 1    | нет операции |
| `ld_a n`    | 0x3e  | 2    | a = n (8‑битное значение) |
| `ld_b n`    | 0x06  | 2    | b = n |
| `add_a_b`   | 0x80  | 1    | a = a + b (обновляет флаги) |
| `jp nn`     | 0xc3  | 3    | безусловный переход на 16‑битный адрес |
| `load_frame`| 0xdf  | 1    | запросить следующий видеокадр (нестандартный опкод, `rst 18`) |
| `halt`      | 0x76  | 1    | останов процессора (завершает поток эмуляции) |

### характеристики (в контексте coredvd)

- 8‑битные данные, 16‑битная адресация (но доступна только память прошивки)  
- переменная длина инструкций: 1, 2 или 3 байта  
- частота выполнения: привязана к частоте кадров (24 гц), один вызов `step_z80()` за кадр  
- нет прерываний, нет ввода-вывода (кроме `load_frame`)  
- флаги обновляются только после `add_a_b`

### пример прошивки

```asm
init:
    ld_a 10       ; a = 10
    ld_b 20       ; b = 20
loop:
    add_a_b       ; a = a + b (30, 50, 70...)
    load_frame    ; показать кадр
    jp loop       ; бесконечный цикл
```

### ограничения и дальнейшее развитие

текущая реализация - минимальная, чтобы продемонстрировать принцип «прошивка управляет плеером».  
в будущем возможно:
- полная эмуляция всех опкодов z80
- подключение реальной памяти (64 кбайт) и инструкций `ld (hl), a`
- поддержка прерываний и системных вызовов через `rst`

пока же z80 в coredvd - это игрушечный процессор, но достаточный для понимания основ.

больше подробностей прошивки в firmware/z80/docs.md

## память и ввод-вывод

- **озу** (ram): 64 кбайт, адреса 0x0000..0xffff
- **vram**: область rgb (320×240×3 байта) доступна только для чтения через графическую подсистему
- **vob-данные**: читаются процессором через регистр r0 (указатель на текущий кадр)
- **порты ввода-вывода** (in/out) пока не реализованы, но зарезервированы на будущее

## сборка и запуск

требования:
- компилятор c++17
- библиотека sfml 2.5+
- python 3.6+

сборка эмулятора:
```bash
g++ -std=c++17 coreDVD.cpp -o coredvd \
    -lsfml-graphics -lsfml-window -lsfml-system \
    -lavformat -lavcodec -lswscale -lavutil \
    -pthread
```

запуск:
```bash
./coredvd
```
эмулятор загружает `firmware.bin` из текущей папки и пытается открыть `VIDEO_TS/VTS_01_1.VOB`.  
если vob нет — генерируется тестовое видео.

## управление в эмуляторе

| клавиша | действие |
|---------|----------|
| m | открыть/закрыть osd-меню |
| esc | закрыть меню |
| пробел | play / pause |
| s | stop |
| ← → | предыдущая / следующая глава |
| d | включить тихую отладку |

## конвертация видео

вы заметили в корне проекта convert.py, так вот объясняю: это скрипт который
конвертирует ваш фильм (input.mp4) в дисковой формат для исполнения программой, 
вот как конвертировать, очень просто на самом деле:

```bash
python3 convert.py
```

только для этого нужно установить ffmpeg

## сборка ассемблеров

так как структура поменялась надо ставить флаг --arch=<процессор> <путь к ассемблеру> <путь к исходному бинару>
вот пример:

```bash
# для z80
python3 asm.py --arch=z80 firmware/z80/firmware.asm firmware/z80/firmware.bin
# для pulse
python3 asm.py --arch=pulse firmware/pulse/firmware.asm firmware/pulse/firmware.bin
```
