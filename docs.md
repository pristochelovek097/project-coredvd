# docs.md

#### о проекте

coredvd — это эмулятор dvd-плееров с собственным 32-битным процессором pulse.  
прошивки для pulse пишутся на ассемблере и компилируются в бинарный файл.  
в будущем в проекте появятся и другие процессоры (z80, arm и т.д.),  
но сейчас основная цель — показать, как работает программируемый dvd-плеер.

**прикол:** процессор реально работает, видео реально крутится, а ты реально можешь написать свою прошивку. или не можешь, но попробовать стоит.

---

## процессор pulse

pulse - это наш risc-подобный 32-битный процессор с 16 регистрами (r0..r15).  
поддерживает флаг z (zero), который устанавливается после `cmp` и некоторых других инструкций.  
стек растёт вниз, указатель стека - r14 (но можно использовать любой, r14 просто договорились).  
программный счётчик (pc) - внутренний, не виден как регистр.

**что умеет:**
- арифметика: add, sub, mul, div, inc, dec
- логика: and, or, xor, shl, shr
- переходы: jmp, jz, jnz, call, ret
- память: mov, movi, load, store, push, pop
- спец: load_frame (сердце плеера), halt, nop

**особенности:**
- `load_frame` — загружает следующий кадр видео и увеличивает r15 на 1
- `r15` работает как счётчик кадров, можешь читать его в прошивке
- все инструкции 32-битные, но `movi` экономит место (16-битный immediate)

полная документация по pulse (с примерами и нюансами) лежит в `firmware/Pulse/docs.md`  
там даже есть пояснение, почему процессор иногда обижается и что с этим делать.

## процессор z80

z80 - это классический 8-битный процессор, который в 80-х крутился в zx spectrum, amstrad cpc и даже в game boy (там похожий).  
в coredvd он реализован в урезанном виде, но достаточном для написания простых прошивок, управляющих видео.

**что реализовано:**
- регистры a (r0), b (r1)
- ld_a, ld_b — загрузка констант
- add_a_b — сложение a и b
- jp — безусловный переход
- load_frame — через RST 0xDF (хитрость эмулятора)
- halt — стоп машина

**чего нет (и не будет):**
- памяти (только регистры, живи с этим)
- условных переходов (кроме jp)
- стека, прерываний, ввода-вывода

**но для простого "крути видео и считай кадры" — за глаза.**

полная документация в `firmware/Z80/docs.md` (там три строчки, не потеряй).

## сборка и запуск

**требования:**
- компилятор c++17 (g++ 7+, clang 5+, или msvc 2019+)
- sfml 2.5+ (графика)
- ffmpeg (libavformat, libavcodec, libswscale, libavutil) — видео
- python 3.6+ (для ассемблера)
- pthread (на linux) или стандартные потоки (на windows)

**linux (просто):**
```bash
g++ src/main.cpp -o coredvd \
    -lsfml-graphics -lsfml-window -lsfml-system \
    -lavformat -lavcodec -lswscale -lavutil -lpthread \
    -std=c++17 -O2
```

**linux (ракета):**
```bash
g++ src/main.cpp -o coredvd \
    -lsfml-graphics -lsfml-window -lsfml-system \
    -lavformat -lavcodec -lswscale -lavutil -lpthread \
    -std=c++17 -O3 -march=native -flto=auto -pipe
```

**windows (mingw, с vcpkg):**
```bash
g++ src/main.cpp -o coredvd.exe \
    -lsfml-graphics -lsfml-window -lsfml-system \
    -lavformat -lavcodec -lswscale -lavutil -lpthread \
    -std=c++17 -O2 -static
```

**windows (msvc):**
```batch
cl /EHsc /O2 src\main.cpp ^
   /I"C:\vcpkg\installed\x64-windows\include" ^
   /link /LIBPATH:"C:\vcpkg\installed\x64-windows\lib" ^
   sfml-graphics.lib sfml-window.lib sfml-system.lib ^
   avformat.lib avcodec.lib swscale.lib avutil.lib ^
   /OUT:coredvd.exe
```

**запуск:**
```bash
./coredvd --cpu=pulse   # по умолчанию
./coredvd --cpu=z80     # для ностальгии
```

эмулятор ищет прошивку в `firmware/<cpu>/firmware.bin`  
и видеофайлы по очереди: `VIDEO_TS/VTS_01_1.VOB` → `movie.mp4` → `video.mkv`  
если ничего не нашёл — показывает летающий логотип и не падает. молодец.

## управление в эмуляторе

| клавиша | действие |
|---------|----------|
| `m` | открыть меню настроек (speed, zoom, osd info, chapter) |
| `v` | открыть меню видеоэффектов (brightness, contrast, scanlines, grayscale) |
| `esc` | закрыть любое меню |
| `пробел` | play / pause (когда видео крутится) |
| `enter` | то же самое (на всякий случай) |
| `s` | stop — возврат в меню диска |
| `←` `→` | предыдущая / следующая глава (300 секунд на главу) |
| `d` | включить тихую отладку (вывод каждые 1000 инструкций) |

**в меню диска:**
- `↑` `↓` — выбор пункта
- `enter` / `пробел` — запуск

**в osd-меню (m):**
- `↑` `↓` — навигация
- `←` `→` — изменение значения (скорость 1/2/4x, zoom on/off, info on/off, chapter +/-)

**в fx-меню (v):**
- `↑` `↓` — выбор эффекта
- `←` `→` — яркость/контраст +/-10%, on/off для scanlines и grayscale

## конвертация видео

в корне проекта лежит `convert.py` — это скрипт, который превращает ваш `input.mp4` в формат, понятный эмулятору.  
точнее, он создаёт `VIDEO_TS/VTS_01_1.VOB` с правильными параметрами:

- разрешение: 720x576 (pal) или 720x480 (ntsc)
- кодек: mpeg2
- звук: пока не поддерживается, но контейнер радуется

**как пользоваться:**

```bash
# установи ffmpeg если ещё нет
# linux:
sudo apt install ffmpeg

# windows:
winget install Gyan.FFmpeg.Essentials

# mac:
brew install ffmpeg

# запускай конвертацию
python3 convert.py
```

скрипт спросит путь к исходному видео или возьмёт `input.mp4` из текущей папки.  
результат — папка `VIDEO_TS` с vob-файлом, который можно скормить эмулятору.

**важно:** vob — это формат dvd-дисков, поэтому эмулятор ищет именно `VIDEO_TS/VTS_01_1.VOB`.  
можно просто переименовать любой mpeg-файл, но лучше использовать скрипт — он выставит правильные параметры.

## сборка ассемблером

теперь структура проекта поменялась, поэтому у ассемблера появился флаг `--arch`.

```bash
# для z80 (если хочется почувствовать себя в 80-х)
python3 asm.py --arch=z80 firmware/z80/firmware.asm firmware/z80/firmware.bin

# для pulse (основной, 32-битный, быстрый)
python3 asm.py --arch=pulse firmware/pulse/firmware.asm firmware/pulse/firmware.bin
```

ассемблер понимает:
- метки (`start:`, `loop:`)
- комментарии (`; это комментарий`)
- числа: десятичные (`10`) и шестнадцатеричные (`0x0A`, `0DEADBEEF`)
- все инструкции процессора (если не понимает — ругается и не компилирует)

**пример прошивки для pulse:**

```asm
start:
    movi r1, 10
    movi r2, 20
loop:
    add r1, r2        ; r1 = r1 + r2
    load_frame        ; показать кадр, r15++
    jmp loop          ; и по новой
```

**пример прошивки для z80:**

```asm
init:
    ld_a 10
    ld_b 20
loop:
    add_a_b
    load_frame
    jp loop
```

после компиляции бинарник кладётся в `firmware/<cpu>/firmware.bin`,  
и эмулятор подхватывает его при запуске с `--cpu=<cpu>`.

## отладка и профилирование

**тихая отладка** (клавиша `d`):
- выводит каждые 1000 инструкций
- показывает pc, текущую инструкцию, значения регистров, флаг z
- полезно, когда прошивка делает не то, что ты ожидаешь (а так бывает часто)

**пример вывода:**
```
[dbg] pc=0x0004 [mov] regs: r0=0x0 r1=0x0 ... z=0
[dbg] pc=0x0008 [add] regs: r0=0xA r1=0x14 ... z=0
[dbg] pc=0x0009 [load_frame] regs: r15=0x1 ... z=0
```

**горячие клавиши в отладке:**
- отладка включается/выключается на лету (клавиша `d`)
- никаких брейкпоинтов пока нет (но можно добавить, если сильно надо)

**профилирование:**
- эмулятор считает количество выполненных инструкций (но не выводит)
- fps отображается в osd-меню (включить `info osd` через `m`)
- при `load_frame` fps ограничен 24 кадрами, скорость регулируется через меню (1x, 2x, 4x)

## частые проблемы и их решение

**проблема:** эмулятор показывает "no disc found", хотя видео есть.  
**решение:** проверь, что файл называется `VIDEO_TS/VTS_01_1.VOB`, `movie.mp4` или `video.mkv`.  
и лежит в той же папке, откуда запускаешь `coredvd`.  
формат mp4/mkv должен быть с h264 или mpeg4 кодеком (ffmpeg поймёт любой, но лучше не экзотику).

**проблема:** прошивка не компилируется.  
**решение:** проверь синтаксис. инструкции должны быть **ровно как в документации**.  
`mov r1 100` не сработает, нужна запятая: `mov r1, 100`.  
и да, регистры только r0..r15.

**проблема:** эмулятор тормозит.  
**решение:** выключи scanlines и уменьши яркость/контраст в fx-меню (`v`).  
или поставь `-O3 -march=native` при компиляции.  
или купи новый компьютер (но это не точно).

**проблема:** при компиляции на windows пишет "sfml-graphics not found".  
**решение:** убедись, что vcpkg отработал: `.\vcpkg list | findstr sfml`  
и не забудь `.\vcpkg integrate install` — это важно.

**проблема:** эмулятор крашится при `load_frame`.  
**решение:** проверь, что видео действительно существует и ffmpeg его понимает.  
запусти `ffmpeg -i video.mp4` — если ошибка, видео битое.

**проблема:** ничего не понятно, но очень интересно.  
**решение:** прочитай `firmware/pulse/docs.md` — там всё разжёвано, с примерами.  
и напиши свою первую прошивку. начни с простого: `load_frame` в цикле. потом добавь счётчик. потом поймёшь, что это весело.
