# docs.md (процессор pulse)

## общие сведения

это документация по **реально работающему** подмножеству процессора pulse в эмуляторе coredvd.  
pulse - 32-битный risc-подобный процессор, но на данный момент реализованы только самые базовые инструкции, необходимые для работы прошивки плеера.

**важно:** не пользуйтесь инструкциями, которых нет в списке ниже - они не сработают и могут обидеть процессор. он и так старается.

## регистры

- **r0 … r15** - 32-битные регистры общего назначения.
- **r15** имеет особую роль: автоматически увеличивается на 1 при каждом успешном вызове `load_frame` (счётчик кадров). прошивка может читать его, чтобы узнать, сколько кадров уже показано. полезно для титров "спустя n кадров..." или просто для скуки.
- все остальные регистры (r0…r14) - свободны, делайте с ними что хотите. ну, почти.
- **флаг z (zero)** обновляется после инструкции `add`. если результат сложения равен 0, флаг z = 1, иначе 0. другие инструкции флаг не меняют - им лень.

## поддерживаемые инструкции

### базовые
- `nop` - ничего не делать
- `halt` - остановить процессор
- `load_frame` - загрузить кадр видео

### перемещение данных
- `mov rd, imm32` - загрузить 32-битную константу
- `movi rd, imm16` - загрузить 16-битную константу
- `load rd, addr` - загрузить из памяти в регистр
- `store addr, rs` - сохранить регистр в память
- `push rs` - положить регистр в стек
- `pop rd` - достать регистр из стека

### арифметика
- `add rd, rs` - сложение
- `sub rd, rs` - вычитание
- `mul rd, rs` - умножение
- `div rd, rs` - деление
- `inc rd` - инкремент
- `dec rd` - декремент

### логика и битовые операции
- `and rd, rs` - побитовое и
- `or rd, rs` - побитовое или
- `xor rd, rs` - побитовое исключающее или
- `shl rd, rs` - сдвиг влево
- `shr rd, rs` - сдвиг вправо

### сравнение и переходы
- `cmp rd, rs` - сравнение (устанавливает флаг)
- `jmp addr` - безусловный переход
- `jz addr` - переход если ноль
- `jnz addr` - переход если не ноль
- `call addr` - вызов подпрограммы
- `ret` - возврат из подпрограммы

реклама авиасейлс...

## системные соглашения

- прошивка загружается по адресу `0x0000`, выполнение начинается с `pc = 0`. всё серьёзно.
- для начала воспроизведения достаточно вызвать `load_frame` в цикле. никакой специальной инициализации регистров не требуется - процессор сам догадается.
- счётчик кадров (`r15`) увеличивается автоматически при каждом вызове `load_frame` в режиме playing. считает как проклятый.
- управление скоростью, паузой, главами и меню осуществляется **с клавиатуры эмулятора**, прошивка за это не отвечает. не надо пытаться сделать это из кода - не выйдет.

## примеры прошивок

### минимальная - бесконечное видео (самое нужное)

```asm
start:
    load_frame
    jmp start
```

### с использованием счётчика кадров (r15) - для умников

```asm
    movi r0, 0
loop:
    add r0, r15       ; r0 = r0 + номер кадра
    load_frame
    jmp loop
```

### демонстрация сложения двух регистров - для задротов

```asm
    movi r1, 10
    movi r2, 20
loop:
    add r1, r2        ; r1 = 10+20 = 30, затем 50, 70, 90...
    load_frame
    jmp loop
```

### продвинутая - с бесконечным циклом и намёком на осмысленность

```asm
    movi r5, 0xfff   ; какой-то счётчик
    movi r6, 1       ; шаг
loop:
    add r5, r6       ; увеличиваем счётчик
    load_frame       ; показываем кадр
    jmp loop         ; и по новой
```

## ассемблер

используйте скрипт `asm.py` из корня проекта. он умный, но не слишком.

```bash
# из папки firmware/pulse/
python3 ../../asm.py --arch=pulse firmware.asm firmware.bin

# или из корня проекта
python3 asm.py --arch=pulse firmware/pulse/firmware.asm firmware/pulse/firmware.bin
```

поддерживаются метки (`start:`), комментарии (`;`), десятичные и шестнадцатеричные числа (`0x10`). постарайтесь не сломать.

## отладка (для смелых)

в эмуляторе нажмите **D** - включится тихая отладка. каждые 1000 выполненных инструкций в консоль будет выводиться:

```
[dbg] pc=0x1234 [mov] regs: r0=0x0 r1=0x41 ... z=0
```

можно смотреть что происходит. или не смотреть, если боитесь правды.

## известные ограничения (читай: "будет в следующей серии")

- нет памяти (ram). только регистры. живите с этим.
- нет условных переходов (`jz`/`jnz`). все прыжки - только безусловные.
- нет вычитания, умножения, деления. только сложение.
- нет логических операций. ни `and`, ни `or`, ни `xor`.
- нет стека. не пихайте туда ничего.
- нет системных вызовов. osd настраивается только с клавиатуры.

## дальнейшее развитие (обещания, которые мы не даём)

в следующих версиях эмулятора планируется добавить:
- память (ram) и операции загрузки/сохранения (`load`/`store`) - когда у процессора появятся карманы
- условные переходы (`jz`/`jnz`) - для принятия решений (это сложно)
- больше арифметики - потому что математика рулит
- системные вызовы для osd - чтобы прошивки могли сами рисовать

пока же pulse реализован ровно настолько, чтобы крутить видео и не падать. и это уже победа.

## часто задаваемые вопросы (faq)

**вопрос:** почему нету процессоров кроме Pulse и Z80
**ответ:** пото пото пото пото потому что это это это это наши чуства люба люба люба люба любовь до безумства

**вопрос:** можно ли использовать r15 для своих целей?  
**ответ:** можно, но он занят подсчётом кадров. не отвлекайте его от работы.

**вопрос:** процессор завис. что делать?  
**ответ:** нажмите esc или закройте окно. не стучите по монитору - ему больно.
