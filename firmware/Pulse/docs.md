# docs.md (процессор pulse)

## общие сведения

эта документация относится к прошивкам для процессора pulse.  
pulse — 32-битный risc-подобный процессор, разработанный специально для coredvd.  
все прошивки должны лежать в этой папке и собираться ассемблером `asm.py` из корня проекта.

## регистры и память

- **r0** — указатель на текущий кадр в vob (32-битное смещение от начала vob-файла)
- **r1..r13** — свободные регистры
- **r14** — указатель стека (sp)
- **r15** — зарезервирован (не используется)
- **флаг z** — обновляется после инструкций add, sub, mul, div, mod, and, or, xor, cmp, inc, dec, lsl, lsr, asr, ror, ldrb, ldrh, pop.

- **ram**: 64 кбайт (0x0000..0xffff). стек растёт вниз, начальное значение sp = 0xffff.
- **vram**: 320×240×3 байта, rgb. пишется только через `load_frame`.
- **vob-данные**: читаются напрямую процессором. текущий кадр — это область yuv размером (320*240*3/2) байт.

## системные соглашения

- при запуске эмулятор загружает прошивку по адресу 0x0000 и начинает выполнение с pc=0.
- перед первым вызовом `load_frame` прошивка должна установить r0 (смещение в vob) и r4 (номер кадра для osd).
- r4 автоматически увеличивается на 1 при каждом `load_frame` в режиме воспроизведения.
- r0 автоматически увеличивается на размер одного кадра (320*240*3/2) при каждом `load_frame`, если включена скорость >1x.
- переключение глав реализуется через запись нового значения в r0 и r4.

## примеры прошивок

### минимальная прошивка (просто показывает видео)

```asm
start:
    mov r0, 0       ; начало vob
    mov r4, 0       ; номер кадра 0
loop:
    load_frame      ; показать кадр
    jmp loop
```

### прошивка с ожиданием нажатия пробела

внимание: так как прямого чтения клавиатуры нет, эмулятор управляет состоянием play/pause по клавише space. прошивка может проверять регистр состояния (в будущем). пока пример просто зацикливает `load_frame`.

### прошивка с выводом символов на консоль

```asm
    mov r1, 'h'
    print_char r1
    mov r1, 'i'
    print_char r1
    halt
```

### системный вызов swi

```asm
    swi 1           ; очистить экран
    swi 0           ; остановить процессор
```

## ассемблер pulse

ассемблер `asm.py` в корне проекта поддерживает все инструкции pulse.

пример сборки:
```bash
python3 ../../asm.py firmware.asm firmware.bin
```

метки, комментарии (`;`), шестнадцатеричные числа (`0x...`) работают как описано в основной документации.

## отладка

в эмуляторе нажмите **d** — включится тихий режим.  
каждые 1000 инструкций процессора в консоль будет выводиться:
```
[dbg] pc=0x1234 [mov] regs: r0=0x0 r1=0x41 ... z=0
```
