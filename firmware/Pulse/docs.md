# docs.md (процессор pulse)

## общие сведения

Это документация по **реально работающему** подмножеству процессора Pulse в эмуляторе coreDVD.  
Pulse — 32-битный RISC-подобный процессор, но на данный момент реализованы только самые базовые инструкции, необходимые для работы прошивки плеера.

**Важно:** Не пользуйтесь инструкциями, которых нет в списке ниже — они не сработают!

## регистры

- **r0 … r15** — 32-битные регистры общего назначения.
- **r15** имеет специальное поведение: автоматически увеличивается на 1 при каждом успешном вызове `load_frame` (счётчик кадров). Прошивка может читать его для получения номера текущего кадра.
- Все остальные регистры (r0…r14) — свободные.
- **Флаг Z (zero)** обновляется после инструкции `add`. Если результат сложения равен 0, флаг Z = 1, иначе 0. Другие инструкции флаг не меняют.

## поддерживаемые инструкции

| инструкция  | синтаксис             | описание |
|-------------|-----------------------|-----------|
| `nop`       | `nop`                 | нет операции |
| `mov`       | `mov rX, imm32`       | загрузить 32-битную константу в регистр `rX` |
| `movi`      | `movi rX, imm16`      | загрузить 16-битную константу (младшие 16 бит, старшие обнуляются) |
| `add`       | `add rX, rY`          | `rX = rX + rY` (32-битное сложение), обновляет флаг Z |
| `jmp`       | `jmp addr32`          | безусловный переход на 32-битный адрес (адрес — метка или число) |
| `load_frame`| `load_frame`          | запросить следующий видеокадр, декодировать, наложить OSD, увеличить `r15` на 1 |
| `halt`      | `halt`                | остановить процессор (завершает поток эмуляции) |

**Никаких других инструкций (`sub`, `mul`, `and`, `or`, `push`, `pop`, `jz`, `call`, `print_char`, `swi`, сдвигов и т.д.) в текущей версии эмулятора НЕТ.** Прошивки, содержащие неизвестные опкоды, будут работать непредсказуемо.

## системные соглашения

- Прошивка загружается по адресу `0x0000`, выполнение начинается с `PC = 0`.
- Для начала воспроизведения достаточно вызвать `load_frame` в цикле. Никакой специальной инициализации регистров не требуется.
- Счётчик кадров (`r15`) увеличивается автоматически при каждом вызове `load_frame` в режиме PLAYING.
- Управление скоростью, паузой, главами и меню осуществляется **с клавиатуры эмулятора**, прошивка за это не отвечает.

## примеры прошивок

### минимальная — бесконечное видео

```asm
start:
    load_frame
    jmp start
```

### с использованием счётчика кадров (r15)

```asm
    movi r0, 0
loop:
    add r0, r15       ; r0 = r0 + номер кадра
    load_frame
    jmp loop
```

### демонстрация сложения двух регистров

```asm
    movi r1, 10
    movi r2, 20
loop:
    add r1, r2        ; r1 = 10+20 = 30, затем 50, 70...
    load_frame
    jmp loop
```

## ассемблер

Используйте скрипт `asm.py` из корня проекта.

```bash
# из папки firmware/pulse/
python3 ../../asm.py --arch=pulse firmware.asm firmware.bin

# или из корня проекта
python3 asm.py --arch=pulse firmware/pulse/firmware.asm firmware/pulse/firmware.bin
```

Поддерживаются метки (`start:`), комментарии (`;`), десятичные и шестнадцатеричные числа (`0x10`).

## отладка

В эмуляторе нажмите **D** — включится тихая отладка. Каждые 1000 выполненных инструкций в консоль будет выводиться:
```
[dbg] pc=0x1234 [mov] regs: r0=0x0 r1=0x41 ... z=0
```

## дальнейшее развитие

В следующих версиях эмулятора планируется добавить:
- память (RAM) и операции загрузки/сохранения (`load`/`store`)
- условные переходы (`jz`/`jnz`)
- больше арифметических и логических инструкций
- системные вызовы для OSD и управления

Пока же Pulse реализован ровно настолько, чтобы крутить видео и демонстрировать принцип программируемого плеера.
